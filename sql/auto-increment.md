# Auto Increment

Automatically generates a unique numerical sequence for a column, typically the **Primary Key**.

## 1. MySQL / MariaDB
```sql
CREATE TABLE Users (
    id INT AUTO_INCREMENT PRIMARY KEY
);
```

## 2. PostgreSQL
```sql
CREATE TABLE Users (
    id INT GENERATED ALWAYS AS IDENTITY PRIMARY KEY -- Modern
    -- id SERIAL PRIMARY KEY                        -- Older
);
```

## 3. SQL Server
```sql
CREATE TABLE Users (
    id INT IDENTITY(1,1) PRIMARY KEY
);
```

## 4. SQLite
```sql
CREATE TABLE Users (
    id INTEGER PRIMARY KEY AUTOINCREMENT
);
```

## Get Last Inserted ID

When you insert a row, you often need its new ID immediately to use as a foreign key.

- **MySQL (`SELECT LAST_INSERT_ID();`)**
  Run immediately after your `INSERT`. Returns the last ID generated by your current connection (safe from race conditions).

- **PostgreSQL (`RETURNING id;`)**
  Add this to the end of your `INSERT` query (e.g., `INSERT INTO Users (name) VALUES ('x') RETURNING id;`). It instantly returns the new ID.

- **SQL Server (`SELECT SCOPE_IDENTITY();`)**
  Run immediately after your `INSERT`. Returns the last ID generated in your current scope, preventing bugs from hidden triggers.
